# 0.5.0

Command ergonomics, preset-first configuration, and Shiki hardening.

## Added

- **`/hunk on` and `/hunk off`.** Auto review pickup now has top-level commands, avoiding nested slash-command completion replacement edge cases.
- **Preset-first `/hunk configure`.** Configure opens to named visual presets (`pi native`, `high-contrast mono`, `warm editorial`, `syntax-only`) with live preview. Advanced keeps the full per-group editor.
- **Picker rows in context.** Symbol and color pickers now render candidates inside representative diff rows instead of abstract glyphs or bare swatches.
- **Lazy Shiki languages.** Non-core languages load on first render and ask the TUI to rerender when ready; common languages remain eager.

## Fixed

- **Nested completion compatibility.** `/hunk auto` completions now insert `auto on`/`auto off` as full arguments, so selecting `on` no longer collapses to an invalid command. `/hunk auto on|off` remains accepted.
- **Configure while streaming.** `/hunk configure` now refuses to open during an active agent response and shows a warning instead of fighting the live transcript renderer.
- **Tokenization guardrails.** Diff tokenization now enforces max-line and time-limit guards, falling back to readable plain text for pathological long lines.
- **ColorReplacements prototype decision.** Full diff tint via Shiki `colorReplacements` is rejected for now: per-side replacements conflict with whole-hunk grammar-state tokenization because add/remove/context rows coexist in one tokenization pass. Side colors remain line furniture and fallback color; Shiki keeps syntax color authority.
